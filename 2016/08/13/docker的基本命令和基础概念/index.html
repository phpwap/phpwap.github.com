docker的基本概念

下图是docker文件系统的一个分层的概念

image

image 是构建 docker 世界的基石。用户基于 image 来运行自己的容器。image 也是 docker 生命周期中的构建部分。

docker image 就是一个只读的模板。
例如：一个 image 可以包含一个完整的 centos 操作系统环境，里面仅安装了 mysql 或用户需要的其它应用程序。

image 可以用来创建 docker container。

docker 提供了一个很简单的机制来创建 image 或者更新现有的 image，用户甚至可以直接从其他人那里下载一个已经做好的 image 来直接使用。

container

container 是基于 image 启动起来的，container 中可以运行一个或多个进程。image 是 docker 生命周期中的构建或者打包阶段，而container 是启动或执行阶段。

docker 利用 container 来运行应用。

container 是从 image 创建的运行实例。它可以被启动、开始、停止、删除。每个 container 都是相互隔离的、保证安全的平台。

可以把 container 看做是一个简易版的 Linux 环境（包括root用户权限、进程空间、用户空间和网络空间等）和运行在其中的应用程序。

注：image 是只读的，container 在启动的时候创建一层可写层作为最上层。

Docker推荐单个容器只运行一个应用程序或者进程，这样就形成了一个分布式的应用程序模式，在这种模式下，应用程序或者服务都可以表示为一系列内部互联的容器，从而使分布式部署应用程序，扩展或者调试应用程序都变得非常简单，同时也提高了程序的内省性。
